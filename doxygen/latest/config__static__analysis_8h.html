Detailed Description

Lazuli configuration file for static analysis.

Copyright
2020, Remi Andruccioli remi..nosp@m.andr.nosp@m.uccio.nosp@m.li@g.nosp@m.mail..nosp@m.com

This file is a redefinition of all the configuration macro constants that are defined in config.h.in. All the macro constants are defined here as if they were true C extern symbols. The reason for that is that most static code analysis tools operate on preprocessed files. By using this file instead of the final config.h, we simulate the fact that these configuration constants can take any value. So static analysizers don't complain/exclude/assume some parts of source code.

Warning
Some of these definitions can cause problems as there are some places in the code where a true constant is expected. e.g when initializing fields of a constant struct. In these case, static analyzers can complain.

◆ LZ_CONFIG_IDLE_TASK_HAS_NAME

const bool LZ_CONFIG_IDLE_TASK_HAS_NAME

Configure if the idle task must have a name.

If set to 1, the idle task will have a name attached to it, and the necessary bytes will be used to store this name in read-only data section. This name is defined in this configuration file.

If set to 0, the idle task will have no name attached to it. Thus it can save a few bytes in the read-only data section.
